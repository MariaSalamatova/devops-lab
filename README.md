**Документація по розробленому веб\-застосунку**

**Task Tracker** — це легковажний RESTful веб\-застосунок, розроблений на базі Python-фреймворку Flask. Автоматизований облік та збереження завдань, застосунок приймає вхідні HTTP-запити через системний сокет (systemd.socket), взаємодіє з базою даних MariaDB для збереження стану та працює за зворотним проксі-сервером (Reverse Proxy) Nginx, який ізолює внутрішні ендпоінти від зовнішнього доступу.

**Функціональні можливості бекенду:** Обробка вхідних HTTP-запитів користувачів, динамічна генерація HTML-сторінок із виводом поточної системної інформації (метрики ВМ, статус) та взаємодія з локальною базою даних з метою збереження й перегляду логів або тестових записів. 

**Продуктивний WSGI-сервер (Gunicorn):** Використовується замість вбудованого сервера Flask для забезпечення стабільної роботи застосунку у production-режимі. Gunicorn підтримує механізм паралельних процесів-воркерів (workers), що дозволяє ефективно утилізувати ресурси процесора. 

**Управління процесами через Systemd (Service \+ Socket):** Flask-застосунок повністю інтегровано в дерево процесів ОС за допомогою створення індивідуального сервісу та сокету. Це забезпечує автоматичний запуск програми під час завантаження системи, перезапуск у разі критичних збоїв, а також реалізацію технології Socket Activation (запуск процесу лише за наявності вхідного мережевого трафіку).

**Реверс-проксі (Nginx):** Виступає в ролі єдиної вхідної точки (Front-end) для всіх зовнішніх запитів на стандартному HTTP-порту 80 . Nginx приймає зовнішній трафік і проксіює його на внутрішній ізольований Unix-сокет системи, що значно підвищує безпеку, швидкість обробки статичного контенту та загальну відмовостійкість системи

**\- Як завантажити/запустити автоматизацію розгортання**  
Переконайтеся, що файли системних конфігурацій юнітів mywebapp.service та mywebapp.socket знаходяться в одній директорії зі скриптом deploy.sh. Надайте скрипту автоматизації права на виконання в системі:   
chmod \+x deploy.sh   
Запустіть процес автоматичного розгортання (скрипт вимагає підвищених прав sudo, оскільки модифікує системні каталоги):  
sudo ./deploy.sh 

**Інструкцію з тестування розгорнутої системи, а саме — як ви перевіряли коректність того, що ви налаштували.**   
Перевірка статусів системних юнітів в утиліті systemd:   
sudo systemctl status mywebapp.socket  
sudo systemctl status mywebapp.service  
Перевірка працездатності та синтаксису веб\-сервера Nginx:  
sudo nginx \-t  
sudo systemctl status nginx  
Локальна перевірка HTTP-запиту безпосередньо всередині ВМ: Виконайте запит до локального веб\-сервера через вбудовану консольну утиліту curl:  
curl \-I [http://localhost/](http://localhost/)
